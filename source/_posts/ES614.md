Map,我对它最深的印象就是在c++里，它可以接受多种数据类型作为键名和键值，而不会受到像JS对象那种键只能是特定类型（字符串）的限制。    
   
## JS中的Map
其实就还是那回事！    
   
```js
'use strict';

let food = new Map();
let fruit = {}, cook = function() {}, dessert = "甜点";

food.set(fruit, "🍋");
food.set(cook, "🍴");
food.set(dessert, "🍩");

console.log(food);
```
如果你在chrome控制台输出，你会看到:   
   
```
Map {Object {} => "🍋", 
function function => "🍴",
"甜点" => "🍩"}
```

哈哈哈哈，简直是对象的究极进化版本！对象也能做键了哈哈哈！！  
   
## Map的api和属性一锅端
常见api们！！   
还拿上面的food开刀吧 ！！   

    
```js
food.size
```
和set一个德性，输出长度！    
    
```js
food.get(fruit);
```
根据键名得到键值。    
    
```js
food.delete(dessert);
```
根据键名删除键值。    
   
```js
food.has(dessert);
```
判断food中是不是有dessert这个键。   
    
```js
food.forEach((value, key) => {
  // todo
});
```
对food中的每个键值对进行操作，value是值， key是键名。    
    
```js
food.clear();
```
清空这个Map!
